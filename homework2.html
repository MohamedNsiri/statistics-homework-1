Explaining the Concept of Dataset and Distribution

Dataset

A dataset is a structured collection of related data that is organized so it can be easily accessed, managed, and analyzed. Think of it as a table in which: Rows represent individual items or observations (for example, each person, transaction, or login attempt). Columns represent the different attributes or variables describing each item (for example, name, age, IP address, or login time).

Distribution

A distribution describes how the values of a variable are spread out or occur within a dataset. It tells you how often certain values appear In statistics, distributions help you understand patterns in data — such as averages, variability, and whether values are concentrated or spread out.

MySQL Database Implementation

Creating the Database and Table

Let's implement our employee computer usage dataset in MySQL:

-- Create the database
CREATE DATABASE company_usage;

-- Use the database
USE company_usage;

-- Create the employees table
CREATE TABLE employees (
  EmployeeID VARCHAR(10) PRIMARY KEY,
  Department VARCHAR(50) NOT NULL,
  HoursUsed INT NOT NULL
);

-- Insert sample data
INSERT INTO employees (EmployeeID, Department, HoursUsed) VALUES
('E001', 'HR', 5),
('E002', 'Finance', 7),
('E003', 'IT', 9),
('E004', 'HR', 4),
('E005', 'IT', 8),
('E006', 'Finance', 6);

Querying the Data for Statistical Analysis

-- Get the univariate distribution for Department
SELECT Department, COUNT(*) AS Frequency
FROM employees
GROUP BY Department
ORDER BY Frequency DESC;

-- Get the univariate distribution for HoursUsed
SELECT HoursUsed, COUNT(*) AS Frequency
FROM employees
GROUP BY HoursUsed
ORDER BY HoursUsed;

-- Get grouped hours range distribution
SELECT
  CASE
    WHEN HoursUsed BETWEEN 0 AND 4 THEN '0-4 hours'
    WHEN HoursUsed BETWEEN 5 AND 7 THEN '5-7 hours'
    WHEN HoursUsed BETWEEN 8 AND 10 THEN '8-10 hours'
  END AS HoursRange,
  COUNT(*) AS NumberOfEmployees
FROM employees
GROUP BY HoursRange
ORDER BY HoursRange;

-- Get bivariate distribution (Department vs Hours Range)
SELECT
  Department,
  SUM(CASE WHEN HoursUsed BETWEEN 0 AND 4 THEN 1 ELSE 0 END) AS '0-4 hours',
  SUM(CASE WHEN HoursUsed BETWEEN 5 AND 7 THEN 1 ELSE 0 END) AS '5-7 hours',
  SUM(CASE WHEN HoursUsed BETWEEN 8 AND 10 THEN 1 ELSE 0 END) AS '8-10 hours',
  COUNT(*) AS Total
FROM employees
GROUP BY Department
ORDER BY Department;

4. Applying Caesar Cipher

The Caesar Cipher is a substitution cipher, it replaces each letter with another that is a fixed number of positions away in the alphabet.

Choosing a shift: 4

Example shifting: A → E, B → F, C → G, D → H

Encrypted Text: HEXEWIGYVMWXMWZIVCSTQTSXMRXMRHISIRVWAQWIXWTVXIGXMRKMRJSVQEXMSRMWIWWIRXMEPTTIVZIRXAEXEO

5. Decrypting Caesar Cipher

Step 1 — Original Text

Before encryption, the paragraph is:

DATA SECURITY IS VERY IMPORTANT IN MODERN SYSTEMS. PROTECTING INFORMATION IS ESSENTIAL TO PREVENT ATTACKS.

After removing spaces and punctuation:

DATASECURITYISVERYIMPORTANTINMODERNSYSTEMSPROTECTINGINFORMATIONISESSENTIALTOPREVENTATTACKS

Step 2 — Encrypted Text (Unknown Shift)

We apply a Caesar cipher with an unknown shift (later discovered to be +4). The encrypted text is:

HEXEWIGYVMWXMWZIVCSTQTVXMRXMRSRIVQWXCQVSXMRKMRJSVQEXMSRMWIWWIRXMEPXSTVIZIRXEXXEGOW

Step 4 — Testing Shift Hypothesis

If we apply a backward shift of 4 (i.e., decrypt by shifting every letter -4 positions):

Step 5 — Fully Decoded Text

DATA SECURITY IS VERY IMPORTANT IN MODERN SYSTEMS PROTECTING INFORMATION IS ESSENTIAL TO PREVENT ATTACKS

Step 6 — Explanation

  • We observed the most frequent letters in the ciphertext.
  • We compared those frequencies with common English letter distributions.
  • We hypothesized that I → E, meaning a shift of 4 backwards.
  • We tested the hypothesis and got readable English — confirming the shift.

Conclusion: Frequency analysis allowed us to decrypt a Caesar cipher without knowing the key.
